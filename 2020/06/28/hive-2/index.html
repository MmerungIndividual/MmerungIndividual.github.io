1、 hive的原子数据类型可以转换吗？如果可以用什么方法转换呢？

6、请创建一个student_ext的外部表含有id，name,sex的信息，以’，’结尾的行格式分隔字段，且数据在hdfs根目录下的/test/stu

7、请问通过create table test1 like test创建的test1和通过
Create table test2 AS SELECT id,name,gfs,adress from test创建的test2的表是否一样，如果不是，请问差别在哪里？
答：不一样。test1只创建表结构，test2不仅创建相应的表结构，并且会插入数据。
8、hive种内部表和和外部表的创建和删除有哪些区别：
答：hive创建内部表时，会将数据移动到数据仓库指向的路径，删除表的时候元数据和数据会被一起删除。
hive创建外部表时，烬记录数据所在的路径，不对数据的位置做任何改变，，删除表的时候只删除元数据，不删除数据。
1：Hive中内部表如何标识，内部表有哪些操作？
内部表创建和传统数据库一样，没有标识关键字；内部表有create、drop、show等操作。

3: create…as….select语法为什么不能应用于外部表？

4：为什么要进行分桶？
取样更高效，以及获得更好的查询处理效率。
5：请写出桶抽样的的语法，并说明参数X和Y分别代表什么含义？
语法：tablesample（bucket x out of y on）y决定抽样比例，即：抽取桶样本个数；x决定起始样本，即：从哪个桶开始选取。

1、聚合函数是一类比较特殊的函数。其可以对多行进行一些计算，然后得到一个结果值。更确切的说，这是用户自定义聚合函数，这类函数中最有名的两个例子是哪两个？分别用于什么。
答：两个例子是count和avg。函数count用于计算有多少行数据（或者某列有多少值），而函数avg可以返回指定列的平均值。

2、当进行算术运算时，需要注意什么问题？请详细说明。
答：需要注意数据溢出或数据下溢问题。Hive遵循的底层Java中数据类型的规则，因此当溢出或下溢发生时计算结果不会自动转换为更广泛的数据类型。乘法和除法最有可能会引发这个问题。

3、与聚合函数“相反的”一类函数是什么函数？它可以做什么？
答：相反的一类函数是表生成函数，它可以将单列扩展成多列或者多行。

4、SELECT语句用于什么？WHERE语句用于什么？两者结合使用又可以做什么？
答：select语句用于选取字段，where语句用于过滤条件，两者结合使用可以查找到符合过滤条件的记录。

5、GROUP BY语句通常会和聚合函数一起使用，怎么分组呢？
答：按照一个或者多个列对结果进行分组，然后对每个组执行聚合操作。

6、Hive可以在map端执行连接过程，Hive对于哪两个连接不支持这个优化。
答：右外连接（RIGHT OUTER JOIN）和全外连接（FULL OUTER JOIN）

7、UNION ALL可以将两个或多个表进行合并，每一个UNION子查询需要具有什么条件？
答：每一个UNION子查询都必需具有相同的列，而且对应的每个字段的数据类型都必须是一致的。

8、对于非常大的数据集，如何满足用户需要的是一个具有代表性的查询结果而不是全部结果？
答：Hive可以通过对表进行分桶抽样来满足这个需求。

9、如果所有表中的数据是分桶的，那么在特定的情况下对于大表同样可以使用map-side JOIN，请简单描述下应满足的条件。
答：简单地说，表中的数据必须是按照ON语句中的键进行分桶的，而且其中一个表的分桶的个数必须是另一个表分桶个数的若干倍。当满足这些条件时，Hive可以在map阶段按照分桶数据进行连接。

10、MapReduce job中传输的所有数据都是按照键值对的方式进行组织的，什么时候必须在内部使用这个功能？
答：Hive在将用户的查询语句转换成MapReduce job时，其必须在内部使用这个功能。
